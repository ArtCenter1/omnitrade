
# CryptoBot Trading Platform - Start Here

Welcome to the CryptoBot Trading Platform project! This document serves as the entry point for anyone contributing to the project, providing essential information about project structure, development practices, and current status.

## Project Overview

This is a full-stack bot trading website with customizable LLM models integration. The platform includes blockchain integration for token distribution, rewarding user referrals and code contributions.

For detailed information about the project, see `project_management/project_overview.md`.

## Project Structure

- `project_management/` - Contains high-level project management files
  - `project_overview.md` - Detailed project description and vision
  - `project_roadmap.md` - Project phases and milestones
  - `current_phase.md` - Current phase status and next steps
  - Phase-specific checklists (e.g., `Phase_1_checklist.md`)

- `reference_files/` - Contains detailed documentation for each feature
  - `index.md` - Index of all reference files with descriptions
  - Feature-specific files (e.g., `trading_bots.md`, `llm_integration.md`)

- `src/` - Source code
  - `components/` - React components
  - `pages/` - Page components
  - `lib/` - Utility functions and helpers
  - `hooks/` - Custom React hooks

## Development Guidelines

### Continuity Between Sessions

To maintain continuity between development sessions:

1. **Check the Current Phase** - Always refer to `project_management/current_phase.md` to understand the current focus and next steps.

2. **Update Reference Files** - When working on a feature, update its reference file with progress, challenges, and next steps.

3. **Follow the Roadmap** - Refer to `project_management/project_roadmap.md` to ensure development aligns with the overall plan.

### Code Standards

- Use TypeScript for all JavaScript code
- Follow the established component structure
- Use Tailwind CSS for styling
- Use shadcn/ui components where applicable
- Implement responsive design for all UI components
- Write descriptive comments and maintain documentation

### Dependencies Management

- All dependencies should be documented and justified
- Major dependencies should be noted in reference files
- Version constraints should be specified and explained
